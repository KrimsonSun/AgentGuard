# 通道 2 · 企业微信语音消息 Agent（设计稿）

> 状态：设计中，Day3+ 实现。核心事实：微信侧**没有实时通话 API**（企微/微信客服只支持语音**消息**），
> 因此本通道是"语音消息一来一回"，不是实时通话——作为呼入的**兜底 + 加分**，非主线。
> 题目点名的 QClaw/OpenClaw 生态（2026-03 腾讯官方接入）验证了这条路"可跑通且稳定"。

## 流程

```
司机扫码加企微客服 → 按住说话发语音 → 客服回调(接收消息API)
  → 下载语音(amr) → STT → 复用 app/ 的同一个大脑(slots/工具/prompt 全套)
  → 回复：文字 + TTS 语音消息 → 收齐后同样 save_visit + notify_guard
```

## 要点

- **复用不重写**：大脑/记忆/台账与电话线完全共用（可插拔呼入边缘，见 HANDOFF §决策3 v2）；
  本目录只做「企微消息回调 ⇄ 音频文件」的适配层。
- 25s 口径对语音消息不完全适用——按"首条语音→微信推送保安"计，答辩说明差异。
- 依赖：企业微信「微信客服」或自建应用的接收消息回调（需可公网回调 URL，可上 CF Workers/Tunnel）。
